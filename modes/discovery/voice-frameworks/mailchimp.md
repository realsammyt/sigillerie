# Mailchimp voice + tone

Source: https://mailchimp.com/resources/voice-and-tone/

## Orientation

Mailchimp's framework splits two things people usually mash together. Voice is the constant. Same character every screen, every page. Tone is the dial. Same brand sounds different in onboarding than in an error state than in a marketing email. The framework treats one brand as one voice with many tones. You write the voice once. You shift the tone per context.

## Voice characteristics

Mailchimp's own list, public reference:

- fun but not silly
- confident but not cocky
- smart but not stodgy
- informal but not sloppy
- helpful but not overbearing
- expert but not bossy
- weird but not inappropriate

For a Sigillerie user, the agent picks 4-6 traits keyed to the user's vibe-words. Each trait gets a "but not" pair so the trait has a wall it cannot climb past. Vibe "irreverent" might land as "playful but not flippant." Vibe "technical" might land as "precise but not robotic."

## Tone shifts table

Same voice. Different dial per context.

| Context | Tone |
|---|---|
| Onboarding | encouraging, patient, low-friction |
| Success state | warm, brief, celebratory but not loud |
| Error state | direct, non-blaming, fix-forward |
| Empty state | inviting, hints at first action |
| Marketing | playful, informative, no pressure |
| Help docs | clear, neutral, lots of nouns and verbs |

## Six example sentences

The agent generates these from intake. Same idea, voice held constant, tone shifted per row.

| Context | Sentence |
|---|---|
| Onboarding | "Welcome. Pick a starter template and we'll fill in the rest." |
| Success | "Saved. Your draft is live in the dashboard." |
| Error | "We couldn't save that. Check the title field and try again." |
| Empty state | "Nothing here yet. Add your first project to get started." |
| Marketing | "New: ship a landing page in an afternoon." |
| Help | "To export, open File then Export. Pick HTML or PDF." |

## When this fits

- Products with broad audience and many touch points
- Customer-service load (help docs, support replies, error states)
- Brands that talk to humans across many emotional registers in one day
- SaaS, e-commerce, consumer apps

## When NOT to use

- Brands that need to feel radical, experimental, or disruptive
- Single-screen products where tone never shifts
- B2B-only products where voice traits like "fun" or "weird" misfire
- Brands with one channel and one register

The Mailchimp framework is for friendly mainstream. If the brand is not friendly mainstream, pick another framework.
